# Kotlin Learning Project

## Instructions

This folder contains files for learning Kotlin in depth. When adding to or expanding this project, follow these guidelines:

- Create subdirectories by topic (e.g., `fun/`, `class/`, `coroutines/`, `collections/`, `types/`, `patterns/`, `gradle/`)
- In each subdir, create `.kt` files that serve the purpose of learning Kotlin in and out
- Do a little bit of the basics, but **mostly focus on advanced stuff**
- Add a lot of inline comments explaining specific APIs and patterns
- Add a `gradle/` folder explaining Gradle concepts
- If a topic is too large for inline comments, add a `.md` file explaining it
- Files should be runnable examples with `main()` functions where possible

## Project Structure

```
kotlin/
├── CLAUDE.md                  ← this file
├── fun/                       ← functions: lambdas, extensions, inline, scope functions
├── class/                     ← OOP: data/sealed/value classes, generics, delegation
├── coroutines/                ← coroutines, flows, channels
├── collections/               ← collection operations, sequences
├── types/                     ← null safety, type system, reified generics
├── patterns/                  ← DSL building, functional patterns, design patterns
└── gradle/                    ← Gradle build system explained
```
